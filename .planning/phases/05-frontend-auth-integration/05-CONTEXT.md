# Phase 5: Frontend Auth Integration — Context

**Gathered:** 2026-03-14
**Status:** Ready for planning

<domain>
## Phase Boundary

React PWA показывает auth-aware UI в рамках одной страницы/приложения (SinodikApp.jsx, без сборки):

- **Неавторизованный:** одна страница — форма записки (поминовения) и справа вверху кнопка/ссылка «Войти». Никаких других вкладок или экранов.
- **Вход:** по «Войти» — один попап с двумя шагами: ввод email → запрос OTP → то же окно переключается на ввод кода → «Войти». Ошибки показываются вверху попапа. Подсказку «Код отправлен на …» не показывать. Кнопки «Назад» на шаге OTP нет.
- **Авторизованный пользователь (не админ):** та же форма записки; «Войти» заменяется на «Выйти» и появляется ссылка «Мои заказы». Отдельный экран «Мои заказы» — список своих записок (пустое состояние — просто пустой список). В карточке минимум: тип, период, дата окончания, имена/количество. Размещение ссылки «Мои заказы» — на усмотрение реализации.
- **Админ:** видит полный интерфейс — все разделы (Сегодня, Поиск, Записка, CSV, Стат., БД). Управление пользователями — в том же месте, где вкладка «БД», но раздел называется **«Податели»** (не «Пользователи»): список подателей со ссылкой на заказы, отфильтрованные по выбранному пользователю. CSV, Стат., БД и вкладки «Сегодня», «Поиск» — только для админа.
- **Выход:** по «Выйти» — сразу сброс токена без подтверждения; после выхода всегда показывается форма записки (гостевой вид).
- **Терминология в UI:** везде «Записки», не «заказы».

Scope: только реализация перечисленного. Новые возможности — в другие фазы.
</domain>

<decisions>
## Implementation Decisions

### Вход (модалка)
- Один попап с двумя шагами подряд: email → OTP в том же окне.
- Ошибки (неверный код, истёкший OTP, сеть) — вверху попапа.
- Подсказку «Код отправлен на …» не показывать.
- Кнопки «Назад» / «Изменить email» на шаге OTP нет.

### «Мои заказы»
- Ссылка ведёт на отдельный экран со списком записок пользователя.
- Пустое состояние — просто пустой список, без текста «Пока нет записок».
- В карточке записки минимум: тип (здравие/упокоение), период, дата окончания, имена или количество имён.
- Размещение ссылки «Мои заказы» в шапке — на усмотрение реализации.

### Админ и видимость разделов
- Управление пользователями в зоне вкладки «БД», раздел называется **«Податели»** (не «Пользователи»).
- На экране «Податели» — только список подателей; у каждой строки — ссылка на заказы этого пользователя (отфильтрованные).
- CSV, Стат., БД, «Сегодня», «Поиск» — только для админа. Обычный пользователь после входа видит только форму записки и «Мои заказы».

### Выход
- По нажатию «Выйти» — сразу сброс токена и показ гостевого вида, без диалога подтверждения.
- После выхода всегда показывается форма записки (главная страница гостя).

### Claude's Discretion
- Точное размещение ссылки «Мои заказы» (справа в шапке, под шапкой и т.п.).
- Детали оформления попапа входа и экрана «Мои заказы» в рамках существующих стилей (T, компоненты).
</decisions>

## Existing Code Insights

### Reusable Assets
- `SinodikApp.jsx`: корневой компонент, состояние `tab`, массив `TABS`, компоненты `Tab`, `AddPage` (форма записки), стили `T`, иконки `Icons`, `api`/`apiOrThrow` без заголовка Authorization — добавить обёртку с Bearer при наличии токена.
- Форма записки уже есть (AddPage); для гостя показывать только её + шапку с «Войти».

### Established Patterns
- Один файл JSX, Babel в браузере, без сборки. JWT в localStorage, отправка через `Authorization: Bearer` (PROJECT.md, REQUIREMENTS FRNT-05).
- Нижняя панель вкладок для админа; для обычного пользователя скрыть панель и показывать только контент формы записки и экран «Мои заказы».

### Integration Points
- Загрузка/проверка токена при старте приложения; GET `/auth/me` для актуальных email/role при наличии токена; при 401 — сброс токена и гостевой вид.
- Все вызовы к `/api/v1/*` и `/auth/*` — через обёртку fetch с подстановкой Bearer и обработкой 401.
- Условный рендер: гость → форма + «Войти»; user → форма + «Мои заказы» + «Выйти»; admin → полный набор вкладок + раздел «Податели».
</code_context>
